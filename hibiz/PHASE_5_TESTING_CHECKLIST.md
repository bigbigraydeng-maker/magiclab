## C · Unsplash 到 Hero 集成测试

### C1 - Unsplash 搜索与"应用到 Hero"功能
- [ ] 进入工具箱，点击「素材库」
- [ ] 在素材库页面顶部，应有 Unsplash 搜索入口
- [ ] 搜索关键词（如 "real estate" 或 "house"）
- [ ] 搜索结果显示 Unsplash 图片
- [ ] **每张图片卡片上应有「⭐ 应用到 Hero」按钮**
- [ ] 点击「⭐ 应用到 Hero」
  - ✓ 确认弹窗/通知显示操作成功
  - ✓ 页面应自动刷新或提示已更新
  - ✓ Hero 被应用的图片应是最新搜索到的图片

### C2 - 既有素材应用到 Hero
- [ ] 在「素材库」页面，查看「我的素材」（已上传的图片）
- [ ] 每张素材卡片上应有「⭐ 应用到 Hero」按钮
- [ ] 选择一张素材，点击「⭐ 应用到 Hero」
- [ ] 确认操作成功

### C3 - Hero 图片在项目页面显示
- [ ] 回到项目详情页 (`/app/projects/[id]`)
- [ ] 在编译卡片或意图输入区域，应显示 **Hero 背景图区块**
- [ ] 区块显示当前已应用的 Hero 图片 **缩略图预览**
- [ ] 区块下方有「从素材库选择或搜索 Unsplash →」链接，点击后跳转 `/media`

### C4 - 网站预览实时更新
- [ ] 仍在项目编辑界面，点击「预览」或切换到预览面板
- [ ] 网站首页的 **Hero 区域应显示新应用的图片**
- [ ] 图片加载正常（不是 404 或加载失败）
- [ ] 切换到其他模块再回到 hero，图片仍然显示

### C5 - Unsplash 归属显示
- [ ] 在网站预览的 footer 或适当位置，应能看到 Unsplash 图片的 **摄影师名字** 和 **Unsplash 链接**（已实现 attribution）
- [ ] 或至少有 Unsplash 品牌标识

### C6 - 多张图片切换测试

- [ ] 回到项目页面预览，确认 Hero 图片已更新为新选择的图片
- [ ] 刷新页面（F5），图片仍然存在（数据持久化）

### C7 - 已发布网站的 Hero 图片
- [ ] 在项目中「发布」或「重新发布」网站
- [ ] 打开公网 URL（microsite URL）
- [ ] 确认公网版也显示更新后的 Hero 图片
- [ ] **注**：如果公网版显示旧图片，可能是缓存；等待几分钟或手动清除浏览器缓存后重试

## E · 错误恢复与边界情况

### E1 - 网络错误恢复
- [ ] 在应用 Hero 图片时，模拟网络延迟或断网
- [ ] 操作应显示错误提示，而非卡死
- [ ] 网络恢复后，再次尝试应该成功

### E2 - 无效 URL 处理
- [ ] 尝试应用一个无效的图片 URL（手动修改 DB 或请求参数）
- [ ] 系统应拒绝或给出清晰错误提示
- [ ] 不应导致网站崩溃

### E3 - 未配置商家信息
- [ ] 创建新项目，不填商家信息
- [ ] 进入工具箱
- [ ] 商家摘要卡片应显示占位符（如 "商家" 或 "-"），不崩溃
- [ ] 所有工具链接仍可用

### E4 - 移动端导航回退
- [ ] 在手机模拟器中，进入工具箱 → 某个子页面（如海报）
- [ ] 点击「返回」链接或浏览器返回，应回到工具箱
- [ ] 再点击返回，应回到项目主页
- [ ] 导航堆栈正确
