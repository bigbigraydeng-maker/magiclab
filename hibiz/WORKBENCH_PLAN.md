# HiBiz — 获客工作台规划（Workbench）

> 锁定日期：2026-04-08  
> 状态：战略方向已确认，实施分阶段推进

## 0. 功能优先级与第三方集成（工程方向）

**优先完成**：在 HiBiz 内交付 **「建站 + 表单」闭环**，能力以 **封装成熟第三方** 为主，自研聚焦 **编排、鉴权、数据回写与合规**，避免从零重做画布或表单运行时。

### 0.1 独立站 / 多页 / 可编辑块

| 方向 | 说明 |
|------|------|
| **目标** | 补 **可编辑块** 与 **多页**，提升与 Wix 类产品的可用性差距，且主数据与发布仍在 HiBiz 可控范围内。 |
| **优先选型** | **Plasmic** 或 **Builder.io**（React/SDK 级集成）；或 **无头 CMS**（如 Sanity、Payload）管内容与路由，前台仍由 Next 渲染。 |
| **刻意少做** | **整站外链型 SaaS**（用户主站完全托管在第三方、仅跳转）— 避免双轨品牌、SEO 与计费割裂。嵌入的是 **块/页级能力**，不是「整站替换 HiBiz」。 |

**已落地（Builder.io，首期）**

- 依赖：`NEXT_PUBLIC_BUILDER_API_KEY`（及可选 `NEXT_PUBLIC_BUILDER_MODEL`，默认 `page`）。见 `.env.example`。
- 公开微站 `/site/[slug]`：在 `microsites.merchant_profile` 中设置 `builder_section_enabled: true` 后，按 Builder 内 **urlPath 定向** 拉取内容；默认 `urlPath` = `/site/{slug}`，可用 `builder_url_path_override` 覆盖；`builder_section_position`：`before`（默认，区块在 `RenderMicrosite` 上方）或 `after`（下方）。
- 在 Builder 控制台为对应 `urlPath` 创建/发布与模型名（默认 `page`）一致的内容后，访客即可看到该区块；未配置 API Key 或未发布内容时不渲染（不影响原有微站）。

### 0.2 表单

| 方向 | 说明 |
|------|------|
| **基座** | 现有 **HiBiz 表单引擎**（`forms` + `submissions` + 蜜罐/限流）保留，作为默认与数据主权路径。 |
| **增强 A** | **Tally** / **Typeform**：**嵌入** + **Webhook** 将提交同步到 HiBiz（或写入 `submissions` 扩展结构），用于高级题型/体验差异化。 |
| **增强 B** | **Formbricks** **自托管**：强品牌化与数据自控，与 A 二选一或分场景并存（实现阶段定策略）。 |

### 0.3 与工作台的关系

获客工作台（里程碑、任务、社媒表等）可在 **建站+表单封装稳定后** 并行或紧随其后迭代，避免同时开三条大线。

---

## 1. 定位

在现有 **微站 + 留资表单 + 轻量数据** 之上，增加 **「以获客战役（Campaign）为中心的工作台」**：把里程碑、模块任务、社媒/SEO 执行清单与 HiBiz 落地页绑定，服务 **澳大利亚与新西兰本地商家**。

**不是**通用 AI Agent 操作系统；**是**可验收的获客交付面 + 封闭工作流 + 必要时封装第三方（测量、素材、发布）。

## 2. 行业优先级

| 阶段 | 行业 | 说明 |
|------|------|------|
| **首期** | **房产中介** | 与现有 TradeMe 提取、房源模块、骨架模板、开放日/留资场景一致；文案与合规围绕物业推广。 |
| **暂缓** | 留学移民中介 | 暂不投入工作台专项；保留代码中已有行业枚举/模板时可不删，产品与市场叙事不再主推。 |

## 3. 首期范围（MVP 建议）

以下按 **价值 / 依赖** 排序，可并行设计，实现按迭代切。

1. **战役容器**  
   - 一个 Campaign 关联现有 `Project`（及 Microsite / Form）。  
   - 里程碑表（节点、时间、说明、输出），只读客户视图（密码、`noindex`）可选后置。

2. **模块任务清单**  
   - 房产中介预设模块：策略与项目、网站优化、追踪与数据、Google Ads、Meta Ads、SEO/GEO、AI 内容辅助、图库与社媒素材等（**先做结构与勾选/状态**，深度 API 后置）。

3. **社媒发帖表**  
   - 周次/槽位、渠道、内容支柱（Educate / Inspire / Convert / Proof）、**主链 → HiBiz 微站 URL**、状态（草稿/已发）。  
   - 与现有海报/分享能力衔接；**API 自动发帖** 非 MVP 必选项。

4. **测量与外链**  
   - UTM 规范、跳转说明；GA4 / Ads 以 **配置向导 + 文档链接** 为先，完整报表后接。

5. **澳洲扩展（数据与话术）**  
   - NZ 优先（现有）；AU 房源/平台链接与本地合规话术逐步补模板，**不阻塞首版 NZ 中介工作台**。

## 4. 与现有架构的关系

- **现有对象**：`User → Project → Intent / Microsite / Form / Submissions`，仪表盘已有访问与转化粗指标。  
- **新增建议**：`Campaign`（或 `Workspace`）作为 **上层容器**，1 Project 可属于 1 Campaign；或 Campaign 即 Project 的扩展字段（由实现阶段决定）。  
- **原则**：骨架与 `assembleRenderModel` **不**为工作台重写；工作台是 **编排与交付层**，落地仍走现有渲染与 RLS。

## 5. 明确不做（首期）

- **整站托管外链**作为主站（与 §0.1 一致）；可视化编辑通过 **SDK/无头 CMS** 集成，不等于自研无限画布。  
- 留学移民垂直的专项合规与模板（暂缓行业）。  
- 对标 Genspark 的「无限工具、多模型闲聊」主界面。
