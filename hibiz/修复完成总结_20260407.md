## 🔧 修复详情

### HIGH-1: 保留商家信息的所有字段

**问题描述**:
用户保存商家信息表单时，以下字段会被**无声地删除**:
- `skeleton_id` (骨架模板选择)
- `theme_overrides` (主题覆盖配置)
- `module_visibility` (模块可见性)
- `property_listings` (房源列表)
- `display_name`, `company_name`, `logo_url`, `avatar_url`, `bio_*` 等所有其他字段

**根本原因**:
原代码仅保留了 3 个字段，其他所有字段都被遗漏:

```typescript
// ❌ 问题代码：只保留 3 个字段，删除其他所有字段
const profile: MerchantProfileV1 = {
  schema_version: 1,
  ...(contact ? { contact } : {}),
  ...(property_promo ? { property_promo } : {}),
  ...(existing?.hero_image_url ? { hero_image_url: existing.hero_image_url } : {}),
};
```

**修复方案**:
先展开所有现有字段，再有选择性地覆盖表单提交的值:

```typescript
// ✅ 修复代码：保留所有现有字段
const profile: MerchantProfileV1 = {
  ...(existing ?? { schema_version: 1 }),  // 保留所有现有字段
  schema_version: 1,
  ...(contact && { contact }),             // 仅覆盖有新值的字段
  ...(property_promo && { property_promo }),
  hero_image_url: existing?.hero_image_url,
};

// 仅当无现有数据且表单为空时才设为 null
const isEmpty = !existing && !contact && !property_promo;
```

**影响**:
✅ 用户可以安全地编辑商家信息表单，不会丢失之前配置的骨架、主题、模块可见性或其他字段

### HIGH-2: 限制外部图片 URL 仅支持 HTTPS

**问题描述**:
`safeExternalImageUrl` 函数接受 `http://` 和 `https://` 两种 URL，导致:
- 浏览器混合内容警告
- 中间人攻击 (MITM) 风险
- 传输中可能被替换图片内容

**根本原因**:

```typescript
// ❌ 问题代码：接受不安全的 HTTP
if (u.startsWith("https://") || u.startsWith("http://")) {
  return u;
}
```

**修复方案**:
仅接受 HTTPS URL:

```typescript
// ✅ 修复代码：仅 HTTPS
if (u.startsWith("https://")) {
  return u;
}
```

**影响**:
✅ 安全性提升。所有外部图片 URL (Unsplash、Supabase Storage) 都经过 HTTPS 验证
✅ 无数据丢失风险，因为 Unsplash 和 Supabase 已使用 HTTPS
